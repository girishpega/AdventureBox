# AdventureBox v0.2.0 — First Playable Adventure

**Release date:** July 12, 2026  
**Theme:** Princess & Unicorn  
**Session length:** 3–5 minutes  
**Age range:** 4–8 years

---

## Summary

v0.2.0 delivers the first complete, playable adventure in AdventureBox. A child can open the box, follow a three-scene story, play a maze mini-game, and earn a celebration sticker — all without reading.

---

## What's New

### Playable Princess Adventure (`/adventure/princess`)

A full four-phase experience:

| Phase | Component | Description |
|---|---|---|
| **Open** | `BoxOpen` | Tap-to-open box animation with sparkles; Princess Lily appears |
| **Story** | `StorySequence` | 3 illustrated scenes with minimal text and large "Next" buttons |
| **Play** | `MazeGame` | Guide the unicorn through a garden maze to find the rainbow star |
| **Celebrate** | `CelebrationScreen` | Confetti, sparkles, trophy, Princess thank-you, sticker reward |

### Reusable Maze Engine (`games/maze/`)

- Deterministic grid-based maze (no random generation)
- Touch/tap and keyboard (arrow keys) controls
- Large 48px+ cells for small fingers
- No timer, lives, score, or failure screen
- Unlimited retries via "Start over"
- Theme-configurable colors and emojis

### Story Configuration (`stories/princess.ts`)

Adventure content is data-driven. Future themes only need to provide:
- Story scenes (text, illustration key, emoji)
- Celebration message and sticker
- Maze grid layout and theme colors

### Sticker Reward

- **Rainbow Helper** 🦄 — awarded on adventure completion
- Displayed with pop animation on celebration screen

## Architecture

### Phase Flow

```
open → story → play → celebrate
         ↑                  │
         └──── Play Again ──┘
```

State is managed locally in `AdventureFlow` via `useState`. No external state libraries.

### Content vs. Engine Separation

| Layer | Location | Swappable per theme? |
|---|---|---|
| Story content | `stories/{theme}.ts` | Yes |
| Maze layout | `stories/{theme}.ts` → `maze.grid` | Yes |
| Maze engine | `games/maze/MazeGame.tsx` | Reused as-is |
| Flow orchestrator | `components/adventure/AdventureFlow.tsx` | Reused as-is |
| Beat components | `components/adventure/*` | Reused as-is |

### Adding a New Theme

1. Create `stories/{theme}.ts` with `PlayableAdventure` config
2. Register in `lib/adventures.ts`
3. Add illustration keys to `StoryIllustration.tsx` (or a theme-specific illustration file)
4. Update `ThemeCard` label if needed

## Accessibility

- Large tap targets (48px minimum on maze cells, 56px on buttons)
- Keyboard navigation via arrow keys in maze
- `aria-label` on interactive maze cells and game container
- `focus-visible` ring styles on all interactive elements
- `prefers-reduced-motion` disables all animations
- Semantic HTML (`header`, `main`, `button`, `role="img"`)

---

## Performance

- Zero new npm dependencies
- All illustrations are inline SVG (no image assets)
- Client components only where interactivity is required
- Static generation for all adventure routes
- Confetti uses deterministic positioning (no `Math.random` at render)

---

## Known Limitations

- Only Princess theme is playable; other themes remain preview-only
- Maze uses tap-adjacent-cell movement (not drag-to-move)
- Story illustrations are placeholder SVGs, not final artwork
- No sound effects or narration
- No child name personalization
- Create beat (coloring) deferred to v0.3
- Sticker is session-only (not persisted between visits)

---

## Suggested Improvements for v0.3

- Drag-to-move maze controls for more tactile feel
- Sound effects (box open chime, success cheer)
- Persist earned stickers in localStorage
- Coloring activity (Create beat)
- Second playable theme (Dinosaurs or Space)
- Child name personalization via parent setup screen
- Final illustrated artwork replacing placeholder SVGs
